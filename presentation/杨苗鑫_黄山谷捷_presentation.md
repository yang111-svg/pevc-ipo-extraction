## 第1页 · 前三周进展总结

### 做了什么

| 周次 | 内容 | 关键产出 |
|------|------|----------|
| **Week1** | PDF获取与解析、建立项目结构 | 8家公司PDF、7步pipeline骨架 |

### 流程全景

```
PDF获取 → parse_pdf(页文本) → locate_sections(定位) → classify_events(分类过滤)
→ extract_candidates(候选块) → LLM提取 → clean/fix → validate_schema → cross_check
                                                                              ↓
                        annotations_pdf ← 关键页批注 ← compare_to_gold(评估)
```

### 核心数字

- **Gold 记录**: 169条（equity_snapshot 31 + subscription_flow 86 + share_transfer_flow 30 + cross_check 22）
- **自动化管线**: 10个步骤，覆盖8家公司
- **annotations_pdf**: 31个关键页批注，25张PNG截图

### 主要困难

1. **PDF表格解析**: 跨页表格合并困难，单元值提取不完整
2. **候选块噪声**: LLM输入前非事件文本混入，导致auto JSONL质量下降
3. **股权转让vs增资**: 同一轮次增资+转让并发，事件边界模糊

### 具体分工

| 步骤 | 方法 | 为什么 |

| 交叉核验 | 规则（流量=存量） | 数学等式，确定性强 |

## 第5页 · Gold 标准：从 PDF 到结构化字段

### 以「吸收合并+股权转让」事件为例

**同一时间点（2021/04/03），两件事需要拆开记录：**

#### ① share_transfer_flow（股权转让）

| 字段 | 值 | 来源 |
|------|-----|------|
| transferor | 昆山谷捷 | PDF直接披露 |
| transferee | 黄山供销集团 | PDF直接披露 |
| transferred_registered_capital_wan | 780.0 | **PDF直接披露**（出资额列） |
| transfer_consideration_wan | 0.0 | PDF披露"零对价" |
| price_per_registered_capital | 0.0 | **须判断：零对价vs价格不明** |
| transfer_type | 零对价转让/吸收合并 | **人工判断**（非直接披露） |

#### ② subscription_flow（吸收合并增资）

| 字段 | 值 | 来源 |
|------|-----|------|
| investor_name | 昆山谷捷（吸收合并） | **须理解事件性质** |
| subscription_qty_wan | 200.0 | 计算：1200-1000 |
| subscription_amount_wan | 0.0 | **须判断：非现金增资** |
| subscription_price | 0.0 | 非现金交易无单价 |

### 哪些字段直接披露，哪些需要计算/判断

| 类别 | 示例字段 | 黄山谷捷例子 |
|------|----------|-------------|
| **直接披露** | shareholder_name, shareholding_pct, 出资额 | 黄山供销78%/780万 |
| **简单计算** | subscription_price = amount/qty | 赛格: 10,377万÷462.8571万=22.42 |
| **需要判断** | event_type（增资vs转让vs吸收合并）, transfer_type | 零对价=架构调整or代持还原？ |
| **须人工核实** | total_shares（PDF加总可能有舍入误差） | 1,804.5113中各股东加总是否闭合 |

## 第6页 · 共性提炼与黄山谷捷特殊难点

### IPO招股书股本变化披露的共性

```
PDF 固定章节结构：
  "发行人基本情况"
    ├── 设立情况（有限公司设立→股份公司设立）
    ├── 报告期内股本和股东变化情况 ← 核心信息来源
    │     ├── 第一次增资/转让
    │     │     ├── 事件描述（自然语言）
    │     │     └── 股权结构表（出资额+比例）
    │     ├── 第二次增资...
    │     └── ...
    └── 发行前股东结构（最终时点快照）
```

- **共性核心页**: 历史沿革章节的"出资额+出资比例"双列表是 Gold 关键来源
- **共性字段**: shareholder_name, shares/出资额, shareholding_pct 几乎总是直接披露
- **共性难点**: 金额单位(万元/元/万股)、跨页表格、股权转让与增资相邻易混淆

### 黄山谷捷的特殊难点

| 难点 | 具体表现 | 影响 |
|------|----------|------|
| **1. 单位口径并存** | PDF同时出现"78%(比例)"和"780万元(出资额)" | 自动化抽取易混淆：78→取哪个？ |
| **2. 零对价转让** | 昆山谷捷→黄山供销/张俊武/周斌，对价0元 | 须判断是"架构调整"而非市场交易，subscription_price=0不代表无意义 |
| **3. 非现金增资** | 吸收合并增资200万，以账面净资产出资无现金流入 | subscription_amount=0但subscription_qty=200，与常规增资逻辑不同 |
| **4. 同一控制下重组** | 股权转让+吸收合并+再增资，三件事在2021年4月密集发生 | 事件分组和时序容易错排 |
| **5. 6家股东→3家→5家→6家** | 股东数量在同一年内反复变化 | 需要正确追溯每步的股东构成 |
